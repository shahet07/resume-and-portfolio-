# User Flow

1. Landing
   - User sees a short explanation and a Start button.

2. Consent + Camera
   - User is asked to enable the camera for sign input.
   - If denied, user can switch to keyboard input.

3. Form Filling
   - User selects a field.
   - User signs in front of the camera.
   - App shows “recognized text” preview.
   - User can Accept, Edit, or Clear.

4. Review
   - Summary of all fields.
   - User can jump back to any field to fix.

5. Submit
   - Prototype shows a confirmation and offers to download or copy the data.
